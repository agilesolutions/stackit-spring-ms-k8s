**Onderwerp:** Verzoek om STACKIT Developer Demo Environment voor Sovereign Cloud Showcase

Beste STACKIT/KPN team,

Mijn naam is Robert Rong en ik ben een ervaren Software Architect en Platform Engineer met meer dan 40 jaar ervaring in softwareontwikkeling, systeemintegratie en cloudplatformen. De afgelopen jaren heb ik mij voornamelijk gespecialiseerd in cloud-native architecturen op Microsoft Azure, Kubernetes, DevOps en Spring Boot microservices.

Momenteel werk ik aan een referentieproject waarmee ik wil aantonen hoe moderne Spring Boot microservices op een veilige en volledig geautomatiseerde wijze kunnen worden ontwikkeld en uitgerold op **STACKIT Sovereign Cloud**.

De demonstratie omvat onder andere:

- Spring Boot 4 microservices (Java 25)
- REST-gebaseerde communicatie tussen microservices
- OAuth2/OpenID Connect beveiliging via Keycloak
- PostgreSQL
- STACKIT Kubernetes Engine (SKE)
- Terraform Infrastructure as Code
- Helm deployments
- FluxCD GitOps
- STACKIT Container Registry
- Prometheus, Grafana, Loki, Tempo en OpenTelemetry
- Complete CI/CD met STACKIT DevOps (GitLab)

De gekozen use case is gebaseerd op een herkenbaar overheidsscenario rondom **Vergunningverlening** en **Zakenregistratie** en is specifiek bedoeld als referentiearchitectuur voor Nederlandse overheidsorganisaties die hun Kubernetes-platform willen migreren van Azure naar een Europese sovereign cloudomgeving.

Op dit moment verblijf ik tijdelijk in Zwitserland. Hierdoor loop ik tegen een praktisch probleem aan: voor het aanvragen van een STACKIT-account wordt een Zwitsers BTW-/VAT-nummer gevraagd, waarover ik niet beschik. Over ongeveer twee maanden keer ik terug naar Nederland, waar ik mij wil richten op opdrachten rondom cloudmigraties naar STACKIT en ondersteuning van overheidsinstellingen bij hun transitie naar een Europese sovereign cloud.

Daarom wil ik graag informeren of het mogelijk is om tijdelijk toegang te krijgen tot een **developer- of demo-omgeving van STACKIT**, zodat ik deze referentiearchitectuur kan afronden en demonstreren.

Het project is nadrukkelijk bedoeld als technisch portfolio en demonstratieomgeving. Ik ben van plan de broncode, Terraform-modules, Helm charts en architectuurdocumentatie openbaar beschikbaar te stellen op GitHub, zodat deze als voorbeeld kan dienen voor organisaties die STACKIT overwegen als cloudplatform.

Ik verwacht dat deze referentiearchitectuur ook voor KPN en STACKIT interessant kan zijn als demonstratie van de mogelijkheden van het platform voor de Nederlandse publieke sector.

Indien gewenst licht ik het project graag verder toe of deel ik de huidige architectuurdocumentatie en de voortgang van het project.

Alvast hartelijk dank voor uw tijd en overweging. Ik zie uw reactie met belangstelling tegemoet.
